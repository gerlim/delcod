# Plano de Implementação da V1 do App de Código de Barras

> **Para agentes de implementação:** SUB-SKILL OBRIGATÓRIO: Use `superpowers:subagent-driven-development` (recomendado) ou `superpowers:executing-plans` para executar este plano tarefa por tarefa. As etapas usam checklist com sintaxe (`- [ ]`) para acompanhamento.

**Objetivo:** Construir a primeira versão orientada à produção de um sistema multiempresa para coleta de códigos de barras, com leitura por câmera no Android, entrada manual na web, administração no desktop, persistência local offline-first e autenticação/sincronização com Supabase.

**Arquitetura:** O aplicativo usa Flutter com MVVM e Riverpod, Drift como armazenamento relacional local offline-first e Supabase para autenticação, persistência em nuvem, RLS e operações administrativas seguras. O comportamento por plataforma é intencionalmente separado: Android concentra a leitura por câmera, Chrome concentra a coleta manual e a exportação, e Windows Desktop concentra administração e auditoria.

**Stack Tecnológica:** Flutter, Dart, Riverpod, Drift, sqlite3/wasm, mobile_scanner, supabase_flutter, excel, pdf, printing, Vercel, Supabase Edge Functions

    return into(readingsTable).insert(entry);

}
